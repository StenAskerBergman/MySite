Process

A series of actions taken in order to achieve my envisioned and desired result

Whenever I'm working with a User Experience, I apply the following steps to improve the quality and properties of that experience leading to a more desired outcome for the stakeholders involved like the business owner and their respective staff plus make the experience more attractive for the user themselves.

Double Diamond

Double Diamond Model orginates from British Design Council and Bela H. Banathy divergence-convergence model(1996), used by IDEO and other Firms.

Troubleshooting & Mapping:

We try to establish a logical, systematic search for the true source of a problem in order to resolve it by addressing it, in our designs. Troubleshooting is needed to identify the symptoms, trace and correct faults in an experience like a game, platform or webshop. A map is a symbolic depiction emphasizing relationships between elements. This depiction contains key values, such as objects, regions, or themes.

Evaluation & Problem Statement:

Evaluation is a systematic determination of a subject's merit, worth and significance, using criteria governed by a set of standards. A problem statement is a concise description of an issue to be addressed or a condition to be improved upon. It identifies the gap between the current state and desired state of a process or product.

Testing & Prototyping:

Conceptualizing a possible solution and creating different variants to test to find the best prototype. If the success rate might be hard to compare or show in which case debate might ensue. Many various testing methods exist to find the next best market solution. A/B Testing is a great one, likewise is usability testing.
